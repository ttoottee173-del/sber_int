# Мотивация участия в проекте

## 1. Почему интересен этот проект

Задача близка к реальному сектору: банки, казначейства, льготные программы — это не учебный пример, а работающий бизнес-процесс с конкретными деньгами и рисками. При этом в исходном состоянии (AS IS) всё держится на ручной проверке и сетевых папках, что само по себе даёт понятное поле для инженерной работы: есть что автоматизировать, и результат автоматизации можно измерить.

Из технической части привлекает связка FastAPI + PostgreSQL + AI-модуль: backend выступает «ядром», к которому потом подключаются ML-компоненты. Это интереснее, чем писать ещё один CRUD, — здесь есть место и для архитектурных решений (версионность, разграничение доступа, обработка неструктурированного ввода), и для интеграции с AI дальше.

## 2. Роль в команде

Вижу себя как backend-разработчика, который закрывает основную серверную часть: API, работа с БД, файловое хранилище. Готов брать на себя скучные задачи тоже — настройку CI, написание тестов, ревью кода — потому что без этого «интересные» задачи быстро превращаются в технический долг.

С командой выстраиваю работу так: договориться о контрактах API заранее, фиксировать решения в документации, не пропадать без предупреждения. Если что-то непонятно по требованиям — спрашиваю, а не додумываю за заказчика.

## 3. Сколько времени готов уделять

Готов выделять порядка 20 часов в неделю на стабильной основе. Период — от 3 месяцев, дальше по обстоятельствам: если проект будет развиваться и задача останётся интересной — готов оставаться дольше.
